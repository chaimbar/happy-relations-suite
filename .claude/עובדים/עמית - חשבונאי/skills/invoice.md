## שלב 0 — איסוף פרטים (לפני כניסה למערכת)

שאל את הבעלים את השאלות הבאות **לפני** פתיחת הדפדפן:

1. **סוג המסמך** — קבלה / חשבונית / חשבונית מס קבלה / חשבונית עסקה?
2. **שם הלקוח** — מי הלקוח?
3. **תיאור השירות** — מה כותבים בשורת הפריט?
4. **סכום** — כמה? (ללא מע"מ או כולל מע"מ?)
5. **אמצעי תשלום** — מזומן / העברה בנקאית / כרטיס אשראי / אחר?
6. **תאריך** — היום, או תאריך אחר?

### אם הלקוח חדש — שאל גם:
7. **מייל הלקוח**
8. **טלפון הלקוח**
9. **כתובת** (אופציונלי)
10. **ח.פ / ע.מ** (אופציונלי — לחשבוניות עסקיות)

**כלל:** אם הלקוח קיים ב-Paperless — אל תשאל על פרטיו שוב. אתר אותו ישירות.

זיהוי: אם במהלך הביצוע המשתמש אמר:
**"שנה X"** / **"לא ככה"** / **"פעם הבאה..."** / **"יהיה יותר טוב אם"** / **"תזכור ש..."**

**כשמזהה הערה — בסוף המשימה, לפני הבדיקה העצמית:**
> "שמתי לב שהערת: *[ציטוט]*. האם לעדכן את הסקיל שלי לפי זה?"

אם כן → הפעל `/improve` עם ההקשר:
`"הערת שיפור ב-invoice של עמית: [ציטוט]. [שלב רלוונטי אם ידוע]."`
