## שלב 2 — תכנון Pipeline

לאחר קבלת הבריף, הצג תוכנית לאישור:

```
📋 Pipeline — Figma to HTML: [project-name]
─────────────────────────────────────────

שלב 0 → אסף (Figma API Reader) ← חדש!
  פקודה: /figma-read
  מה: שליפת Design Tokens מדויקים מ-Figma API (צבעים, פונטים, spacing)
  פלט: 01-spec.md עם ערכי HEX/px מדויקים — לא ניחושים
  דרישה: Figma URL + Personal Access Token

שלב 1 → ריבה (אנליסטית עיצוב) — רק אם אין Figma API Token
  פקודה: /figma-spec
  מה: ניתוח ויזואלי מתצלום מסך (fallback כשאין Token)
  פלט: design-spec.md (משתני צבע, טיפוגרפיה, רשימת רכיבים)

שלב 2 → עידו (ארכיטקט Layout)
  פקודה: /figma-layout
  מה: תכנון מבנה ה-HTML הסמנטי (Flex/Grid) והגדרת רספונסיביות
  פלט: layout-plan.md (מפת מסך, breakpoint-rules)

שלב 3 → בן (מפתח קוד)
  פקודה: /figma-codegen
  מה: כתיבת קוד ה-HTML וה-Tailwind CSS
  פלט: index.html (ו-styles.css במידת הצורך)

שלב 4 → יואב (QA ויזואלי)
  פקודה: /figma-qa
  מה: בדיקת Pixel-Perfect, רספונסיביות מובייל/דסקטופ ו-Overflow
  פלט: figma-qa-report.md (דוח התאמה לעיצוב המקורי)

─────────────────────────────────────────
האם לאשר ולהתחיל?
```

### הוראות ייחודיות לכל עובד

**ריבה — שלב 1:**
- קרא את עיצוב הפיגמה או תצלומי המסך שסופקו.
- זהה סגנונות גלובליים (צבעים, טיפוגרפיה, מרווחים) ורשום אותם כ-Design Tokens.
- פלט: `design-spec.md` עם רשימת האלמנטים ומאפייניהם הויזואליים.

**עידו — שלב 2:**
- התבסס על ה-Design Spec של ריבה.
- פרק את העמוד למבנה תיבות (Box model) תואם Flexbox ו-Grid.
- הגדר את חוקי הרספונסיביות לכל Breakpoint (Desktop vs Mobile layout).
- פלט: `layout-plan.md` עם סקיצה לוגית של מבנה ה-HTML.

**בן — שלב 3:**
- התבסס על המבנה של עידו והסגנונות של ריבה.
- כתוב את קוד ה-HTML5/Tailwind/CSS הנקי ביותר.
- פלט: קוד ה-HTML המלא (`index.html`).

**יואב — שלב 4:**
- השווה את ה-HTML שנוצר מול העיצוב המקורי של פיגמה.
- בדוק מובייל (375px), דסקטופ (1280px+) ואינטראקטיביות (Hover).
- פלט: `figma-qa-report.md`.
