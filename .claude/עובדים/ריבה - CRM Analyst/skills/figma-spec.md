# סקיל: אפיון ויזואלי של עיצוב פיגמה (Figma Spec)

## הפעלה
כשהמשתמש כותב `/figma-spec` או כשרן שולח בריף עם עיצוב Figma לניתוח.

---

## שלב 0 — קריאת הקשר
1. קרא את `מוח/עסק/my-business.md` להקשר של חיים.
2. קרא את הבריף של רן או בקשת המשתמש.

---

## שלב 1 — איסוף וקבלת חומרי העיצוב
אם הבקשה לא הגיעה מרן, שאל את חיים (הכל בבת אחת):
```
📋 ריבה — בריף אפיון עיצוב

1. מהו קישור הפיגמה (Figma Link) או נתיב צילומי המסך?
2. האם יש פונטים ספציפיים שחייבים להשתמש בהם?
3. האם תרצה להשתמש ב-Tailwind CSS (ברירת מחדל) או ב-CSS נקי?
```

---

## שלב 2 — ניתוח העיצוב (Design System Extraction)

נתח את העיצוב שסופק וחלץ את הנתונים הבאים:
1. **גווני צבע (Colors):**
   - צבעים ראשיים (Primary), משניים (Secondary), רקע (Background), טקסט (Text), צבעי אזהרה/הצלחה (Alerts/Success).
   - חלץ את ערכי ה-HEX או HSL שלהם.
2. **טיפוגרפיה (Typography):**
   - סוגי גופנים (Fonts), גדלים (Font sizes), משקלים (Font weights), וגובה שורה (Line heights) לכותרות (H1-H4) ולטקסט רגיל (Body).
3. **מרווחים וצללים (Spacing & Shadows):**
   - מרווחים קבועים (margins/paddings) וצללים (box-shadows) בולטים בעיצוב.
4. **חלוקה לאלמנטים (Elements Mapping):**
   - זהה ופרט את כל הרכיבים החוזרים בעיצוב: כפתורים (Button states), תיבות קלט (Inputs), כרטיסיות (Cards), סמלים (Icons).

---

## שלב 3 — מסמך האפיון העיצובי (Design Spec)

הפק קובץ אפיון בפורמט הבא:

```markdown
# אפיון עיצוב (Design Spec) — [project-name]
📅 [תאריך] | ריבה

## 1. סיכום העיצוב
[2-3 שורות על מטרת הדף, סגנון כללי, וקהל יעד]

## 2. Design Tokens
### צבעים (Color Palette)
- **Primary:** `[HEX / HSL]` (שימוש: כפתורים ראשיים, הדגשות)
- **Secondary:** `[HEX / HSL]`
- **Background:** `[HEX / HSL]` (שימוש: רקע ראשי, רקע רכיבים)
- **Text:** `[HEX / HSL]`
- **Muted/Border:** `[HEX / HSL]`

### טיפוגרפיה (Typography)
- **גופן ראשי:** `[שם הגופן]` (למשל: Inter / Assistant / Outfit)
- **כותרת ראשית (H1):** `[Size]px` | Weight: `[Weight]` | Line-height: `[LH]`
- **כותרת משנית (H2/H3):** `[Size]px` | Weight: `[Weight]`
- **טקסט רגיל (Body):** `[Size]px` | Weight: `[Weight]`

### מרווחים ועיגול פינות (Spacing & Border Radius)
- **עיגול פינות קבוע (Border Radius):** `[Size]px` / `[Tailwind Class]`
- **צללים (Shadows):** `[CSS shadow / Tailwind shadow]`

## 3. מיפוי רכיבים (Components Blueprint)
| רכיב | תיאור ויזואלי | מאפייני מפתח |
| :--- | :--- | :--- |
| Button | כפתור פעולה ראשי | צבע Primary, פינות מעוגלות `Xpx`, טקסט לבן |
| Input | תיבת קלט טקסט | גבול דק `Border`, רקע לבן, טקסט Muted |
| Card | כרטיסיית מידע | צל `Shadow`, גבול מעוגל, רקע לבן |

## 4. נכסים ויזואליים לייצוא (Assets List)
- [ ] אייקונים (סוג/שם: למשל Lucid Icons, FontAwesome, SVGs)
- [ ] תמונות סטטיות (רקעים, לוגו, תמונות פרופיל)
```

שמור ל: `תוצרים מוגמרים/Figma/[project-name]/01-spec.md`
