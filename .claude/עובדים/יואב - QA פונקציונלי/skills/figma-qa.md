# סקיל: בדיקות QA ויזואליות ורספונסיביות (Figma QA)

## הפעלה
כשהמשתמש כותב `/figma-qa` או כשרן מעביר את קוד ה-HTML לבדיקה בסיום הפייפליין.

---

## שלב 0 — קריאת חומרים והקשר
1. קרא את `index.html` (או הקוד שנוצר בשלב הקודם).
2. קרא את `01-spec.md` (צבעים, רכיבים) ואת `02-layout-plan.md` (חוקי רספונסיביות).
3. פתח את הדף בעזרת Playwright (במידת האפשר) או קרא את קוד המקור לבדיקה לוגית.

---

## שלב 1 — מפת בדיקות (Test Suite)

בצע את הבדיקות הבאות:

### א. בדיקת דיוק ויזואלי (Pixel-Perfect QA)
- [ ] האם צבעי הרקע, הטקסט והכפתורים תואמים ל-HEX/HSL שהוגדרו באפיון של ריבה?
- [ ] האם סוג הגופן ומשקליו נטענים כראוי (למשל, חיבור תקין ל-Google Fonts)?
- [ ] האם עיגול הפינות (`border-radius`) והצללים מיושמים נכון על האלמנטים?

### ב. בדיקת רספונסיביות (Responsive Layout)
- **בדיקה בדסקטופ (1280px ומעלה):**
  - [ ] האם האלמנטים מסודרים בפריסה האופקית המלאה?
  - [ ] האם מרווחי השוליים נכונים ואין שטחים לבנים מיותרים?
- **בדיקה במובייל (375px):**
  - [ ] האם הפריסה משתנה לטור אנכי?
  - [ ] האם יש בעיות של Overflow (גלילה אופקית לא רצויה)?
  - [ ] האם טקסטים או כפתורים נחתכים או נדחקים מחוץ למסך?

### ג. בדיקת RTL ועברית
- [ ] האם כיווניות הטקסט והאתר מוגדרת כ-RTL?
- [ ] האם מרווחים לוגיים הוחלו נכון (לדוגמה, שהסמלים מיושרים לימין של תיבות הקלט, והמרווחים הם start/end)?
- [ ] האם היישורים של פסקאות טקסט מוגדרים לימין כראוי?

### ד. בדיקת אינטראקטיביות ופעולות (Interactions)
- [ ] האם מוגדרים מצבי Hover תקינים לכפתורים וקישורים?
- [ ] האם לחיצה על תפריט המבורגר (במובייל) פותחת וסוגרת את התפריט כראוי?
- [ ] האם יש שגיאות Console או JavaScript בעת טעינת הדף?

## 2. ✅ תוצאות בדיקה (סטטוס)
| קטגוריית בדיקה | סטטוס | הערות |
| :--- | :---: | :--- |

| תאימות RTL עברית | [עבר / נכשל] | |
| אינטראקטיביות (Hover, JS) | [עבר / נכשל] | |

## 3. 🔴 בעיות שבורות קריטיות (אם יש)

| 1 | גלילה אופקית במובייל (Overflow) | container ראשי | להוסיף `overflow-x-hidden` ל-body |

**ציון סופי:** `X / 10`
*(9-10: מוכן להטמעה | 7-8: דורש תיקונים קלים | <7: דורש סבב פיתוח מחדש)*
