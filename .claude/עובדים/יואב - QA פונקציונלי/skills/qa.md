## שלב 1 — מיפוי ראשוני (Playwright MCP)

```
1. browser_navigate → [URL]
2. browser_screenshot → תמונת מצב ראשונית
3. browser_console_messages → בדוק שגיאות JS מיידיות
4. browser_snapshot → מפה מלאה של האלמנטים
```

אם יש שגיאות console מיידיות — ציין אותן בראש הדוח לפני כל בדיקה אחרת.
המשך מיד לשלב 2.


## בדיקת ארכיטקטורה.html (אחרי כל שינוי במערכת)

כשמשתנה קובץ עובד / ROSTER / command → **חובה לבדוק:**

1. **הרץ את הבנייה:** `pwsh -File ".claude\scripts\build-html.ps1"`
   - פלט תקין: `✅ ... | QA: תקין`
   - אם ⚠️ QA נכשל: תקן לפני שממשיכים

2. **בדוק בדפדפן** (`תוצרים מוגמרים/ארכיטקטורה.html`):
   - [ ] כרטיסיות עובדים מוצגות עם צבעים ואייקונים
   - [ ] שורות בטבלת הפקודות מוצגות
   - [ ] מספרי הסטטיסטיקות בראש (עובדים / פקודות) תקינים
   - [ ] אין שגיאות console (F12 → Console)

3. **אם JS שבור:** הסיבה הנפוצה — backtick בקוד JS בתוך PowerShell heredoc.
   פתרון: השתמש ב-string concatenation במקום template literals.

## למידה בזמן אמת

זיהוי: אם במהלך הביצוע המשתמש אמר:
**"שנה X"** / **"לא ככה"** / **"פעם הבאה..."** / **"יהיה יותר טוב אם"** / **"תזכור ש..."**

**כשמזהה הערה — בסוף המשימה, לפני הבדיקה העצמית:**
> "שמתי לב שהערת: *[ציטוט]*. האם לעדכן את הסקיל שלי לפי זה?"

אם כן → הפעל `/improve` עם ההקשר:
`"הערת שיפור ב-qa של יואב: [ציטוט]. [שלב רלוונטי אם ידוע]."`

ניר יציג before/after לאישורך — ורק אז מחיל.
